Trama Saga Orchestrator

Coordinate multi-service workflows with retries and compensation. Trama provides a simple HTTP API, persistent execution tracking, and operational visibility out of the box.

Get Started Read README OpenAPI

Why teams use Trama

  • Versioned saga definitions and runtime execution API.
  • Retry/backoff policies plus compensation workflows.
  • Execution persistence for auditing and diagnostics.
  • Metrics + tracing for production operations.
